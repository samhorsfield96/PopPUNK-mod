# PopPUNK-mod
PopPUNK-mod is used for fitting models of pangenome dynamics to PopPUNK pairwise core vs. accessory distance distributions.

PopPUNK-mod uses Approximate Bayesian Computation (ABC) using [BOLFI](https://jmlr.org/papers/v17/15-017.html), part of the [ELFI](https://www.jmlr.org/papers/volume19/17-374/17-374.pdf) package.

## Installation

First install all dependencies. This is easiest using [micromamba](https://mamba.readthedocs.io/en/latest/installation/micromamba-installation.html).

```
micromamba env create -f environment.yml
micromamba activate pp_mod
```

This will create and activate the pp_mod environment and takes 5-10 minutes to install. This describes all versions of software used for the [PopPUNK-mod paper](https://www.biorxiv.org/content/10.64898/2026.04.28.721069v2). Then, clone this repository:

```
git clone https://github.com/samhorsfield96/PopPUNK-mod.git && cd PopPUNK-mod
```

Finally, install [Pansim](https://github.com/bacpop/Pansim), a fast pangenome dynamics simulator used by PopPUNK-mod. Installation instructions can be found on the Pansim [Github](https://github.com/bacpop/Pansim).

## Running Pansim

You can run a wrapper around Pansim using `pansim.py`, ensuring you point to the Pansim executable:

```
python pansim.py --pansim_exe /path/to/pansim --outpref simulations
```

### Outputs

This will generate:
- All outputs generated by Pansim (see [repository](https://github.com/bacpop/Pansim))
- `<outpref>_sim.png`; a graph showing the distances, along with a asymototic curve fit

## Running PopPUNK-mod

PopPUNK-mod requires a target distribution to fit to, as well as at least one parameter to fit. All other parameters can be fixed.

`--param` arguments are those to be fitted, whilst `--fixed-param` are those to be fixed. Those not specified will take default parameters in Pansim. We advise setting all parameters to ensure reproducibility. Names used must be identical to arguments in Pansim.

```
python poppunk_mod.py --pansim_exe /path/to/pansim --outpref predictions --param rate_genes1 0.01 1.0 loguniform --param prop_genes2 0.0 1.0 uniform --fixed-param pan_genes 2000 --fixed-param pop_size 1000
```

In the above example, two fitted and two fixed parameters are specified. 

For the fitted parameters, syntax follows the `--param NAME MIN MAX DIST`:
- NAME: rate_genes1, MIN: 0.01, MAX: 1.0, DIST: loguniform
- NAME: prop_genes2, MIN: 0.0, MAX: 1.0, DIST: uniform

where `NAME` matches an argument in Pansim, `MIN`/`MAX` define the parameter value search range, and `DIST` defines the prior distribution to use, one of `uniform` or `loguniform`.

For the fixed parameters, syntax follows the `--param NAME VALUE`:
- NAME: pan_genes, VALUE: 2000
- NAME: pop_size, VALUE: 1000

where `NAME` matches an argument in Pansim and `VALUE` defines the fixed parameter value.

### Outputs

PopPUNK-mod generates a number of outputs:
- directory named `<outpref>`: contains simulation samples from run, can be loaded again using `--load`.
- `<outpref>_parameter_estimates_summary.csv`: csv containing parameter mean, median and 95% credible interval values.
- `<outpref>_ELFI_summary.txt`: csv containing normalised parameter mean, median and 95% credible interval values.
- `<outpref>_gp_evidence.csv`: csv containing parameter values and associated discrepancy values from BOLFI sampling.
- `<outpref>_mcmc_posterior_samples.csv`: csv containing MCMC sampling values for posterior after burn-in.
- `<outpref>_pairs.png`: plot of paired parameter posterior distributions.
- `<outpref>_BOLFI_discrepancy.png`: plot of discrepancy values from BOLFI sampling.
- `<outpref>_BOLFI_traces.png`: plot of MCMC chains.
- `<outpref>_contours.png`: plot of target pairwise core vs. accessory distance distributions with density contours.
- `<outpref>_curve_fit.png`: plot of target pairwise core vs. accessory distance distributions with negative exponential curve fit.
- `<outpref>_marginals.png`: marginalised parameter posterior distributions.

## An example

In this guide we will conduct a run of PopPUNK-mod. Before starting, please follow the installation instructions above.

1. First we require a fitted PopPUNK model. Either generate your own fit following the instructions [here](https://poppunk-docs.bacpop.org), or download a fit from the [Bacpop](https://bacpop.org/poppunk-databases/) website. For this example you can use the data [here](./example_input).

2. Once you have your fit, extract distances from them using the [extract_poppunk_distances.py](./scripts/extract_poppunk_distances.py), referring to the `dists` files generated by your PopPUNK run. Alternatively, there are example distances available [here](./distances/2_column). 

3. Now we need pangenome size estimates. These can come from your favourite pangenome clustering tool. Alternatively, you can run [WTBcluster](https://github.com/samhorsfield96/WTBcluster) on the genomes you used in Step 1, which will generate a file called `tokenised_genomes_0.txt` that can be passed to [analyse_pangenome.py](./scripts/analyse_pangenome.py), which will generate all the statistics you need to run PopPUNK-mod. For the example [here](./example_input), we provide them in the `pangenome_summary.tsv` file and are updated below.

4. Now you're ready to run PopPUNK-mod, specifying your fixed and free parameters. To recreate the analysis in the PopPUNK-mod paper, the add this to a bash script and run:

```
# pangenome parameters e.g. from WTBCluster
pan_genes=5000 #[total pangenome size]
core_genes=100  #[number of core genes, frequency >=95%]
intermediate_genes=1139 #[number of core genes, 5% <= frequency < 95%]
rare_genes=2666 #[number of core genes, frequency < 5%]
avg_gene_freq=0.25  #[average proportion of pangenome found in each genome]

# default params
genome_size_penalty=0.99
covar_scaling=0.1
prop_positive=-0.1
neg_lambda=100.0
pos_lambda=100.0
competition_strength=0
HR_rate=0.0
HGT_rate=0.0
core_threshold=0.95
rare_threshold=0.05
prop_positive_min=-0.1
rate_genes2=10.0
core_nuc=$((core_genes * 1000))
n_gen=500
pop_size=1000

# not estimated
fixed_params="--fixed-param pop_size ${pop_size} --fixed-param core_size ${core_nuc} --fixed-param pan_genes ${pan_genes} --fixed-param core_genes ${core_genes} --fixed-param avg_gene_freq ${avg_gene_freq} --fixed-param n_gen ${n_gen} --fixed-param prop_positive \"${prop_positive}\" --fixed-param rate_genes2 ${rate_genes2} --fixed-param pos_lambda ${pos_lambda} --fixed-param neg_lambda ${neg_lambda} --fixed-param HR_rate ${HR_rate} --fixed-param HGT_rate ${HGT_rate} --fixed-param genome_size_penalty ${genome_size_penalty} --fixed-param competition_strength ${competition_strength}"

# to be estimated
fitted_params="--param rate_genes1 1e-5 1.0 loguniform --param core_mu 1e-9 1e-3 loguniform --param prop_genes2 0.0 1.0 uniform"

# to run PopPUNK-mod
python poppunk_mod.py --pansim_exe /path/to/pansim --distfile ./example_input/core_mu_1e-5_rate_genes1_1e-3_prop_genes2_0.125.tsv --core-threshold ${core_threshold} --rare-threshold ${rare_threshold} --gene-counts "${core_genes},${intermediate_genes},${rare_genes}" --outpref outprefix --threads 10 --max_distances 100000  --samples 100000 --init_evidence 150 --n_evidence 600 --update-int 1 --chains 4 --acq-noise-var 0.01 --kernel RBF --gamma 0.25 --workdir tmp_dir --covar-scaling ${covar_scaling} ${fixed_params} ${fitted_params}
```

5. PopPUNK-mod will now run for 1-2 days using 16 threads, depending on how complex the likelihood surface is of your dataset (this is likely impossible to know at the start). Upon finishing, you'll get a set of figures and tables (examples can be found [here](./example_outputs/))

- `_contours.png`: core vs. accessory distances with contours.
- `_curve_fit.png`: exponential decay curve fit to distances.
- `BOLFI_traces.png`: MCMC traces from BOLFI - these should look like "caterpillars" for each parameter if fitting is done correctly that don't span fully 0-1. If not, the model is likely non-identifiable from the parameters chosen.
- `_BOLFI_discrepancy.png`: BOLFI sampling, there should be a single upside-down peak around the optimal parameter estimate.
- `_ELFI_summary.txt`: Table of parameter estimates. NOTE: these require back-transforming from 0-1 range.
- `_gp_evidence.csv`: Summary table of data plotted in `_BOLFI_discrepancy.png`
- `_marginals.png`: Marginal likelihood distributions of parameters.
- `_mcmc_posterior_samples.csv`: all posterior samples accepted by BOLFI.
- `_pairs.png`: paired marginal distributions
- `_parameter_estimates_summary.csv`: back-transformed parameter estimates.
